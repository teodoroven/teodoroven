## Ключевые проекты

### Edo — автоматизация ЭДО и OCR документов  
_Приватный коммерческий проект_

Система для автоматизации электронного документооборота. Проект обрабатывает документы из СБИС по API, а также бумажные документы и PDF/сканы через OCR. Результаты синхронизируются в Google Таблицы, чтобы заказчик мог работать с данными вручную и видеть актуальное состояние.

**Стек:**  
Python, СБИС API, PaddleOCR, Tesseract, PyMuPDF, Pillow, SQLite, Google Sheets API, Google Drive API, ThreadPoolExecutor, PyInstaller, unittest

**Что реализовано:**

- интеграция со СБИС API
- загрузка входящих и исходящих документов
- скачивание и отслеживание статусов вложений
- OCR-обработка PDF, изображений и сканов
- многоэтапное определение типа документа
- классификация бухгалтерских документов
- извлечение ИНН, дат, номеров, контрагентов и статусов
- проверка дубликатов
- локальное хранение состояния в SQLite
- синхронизация результата в Google Таблицы
- пакетные обновления через Google Sheets API
- обработка ошибок и повторные попытки при сбоях
- упаковка приложения в standalone `.exe`
- тесты для ключевых частей логики и OCR-сценариев

**Особенность проекта:**  
важен не только внутренний результат программы, а актуальная информация в Google Таблице, с которой продолжает работать заказчик. Программа учитывает ручные изменения в таблице и синхронизирует данные так, чтобы Google Sheets оставался основным рабочим интерфейсом бизнеса.

### [Parser / MailParser](https://github.com/teodoroven/MailParser) — мониторинг почты и Google Sheets

Desktop-приложение для автоматического мониторинга входящих писем по IMAP, анализа писем по пользовательским правилам и записи результата в Google Таблицы.

Пользователь настраивает почту, credentials Google API, правила-триггеры, листы таблицы и расписание через графический интерфейс. После этого приложение вручную или по расписанию читает письма, применяет правила, извлекает нужные данные и обновляет Google Таблицу.

**Стек:**  
Python, Eel, Vanilla JavaScript, HTML, LESS, IMAP, Google Sheets API, threading, pystray, tkinter, unittest

**Что реализовано:**

- подключение к почте через IMAP
- анализ тем и текстов писем по пользовательским триггерам
- извлечение дат, контрагентов, периодов и типов отчётности
- запись результата в Google Таблицы
- цветовое кодирование и комментарии в ячейках
- графический интерфейс на Python + Eel
- динамический UI на Vanilla JavaScript
- настройка триггеров и расписаний без редактирования кода
- фоновая работа через системный трей
- хранение и шифрование настроек
- многопоточная синхронизация без блокировки интерфейса
- обработка ошибок сети и внешних API
- тесты для ключевой бизнес-логики
- подготовка Windows-сборки приложения

**Особенность проекта:**  
Google Таблица используется как основное рабочее пространство заказчика: пользователь может вносить изменения вручную, а программа учитывает актуальное состояние таблицы. Это не просто выгрузка данных, а синхронизация с рабочим инструментом, которым заказчик продолжает пользоваться каждый день.

### [Bots](https://github.com/teodoroven/Bots) — Telegram/VK bot framework для автошколы

Мультиплатформенная bot-система для автошколы с поддержкой Telegram и ВКонтакте, пользовательскими и административными сценариями, записью клиентов, связью клиента с администратором и GPT-ответами с настраиваемыми лимитами, контекстами и моделями.

Проект построен как приложение по слоям: transport-layer скрывает различия Telegram/VK, app-layer реализует бизнес-сценарии и state-machine сессий, а storage-layer хранит состояние через SQLAlchemy/Alembic.

**Стек:**  
Python 3.12, Telegram Bot API, VK API, OpenAI-compatible API, tiktoken, SQLAlchemy, Alembic, SQLite/PostgreSQL, Docker Compose, unittest, logging

**Что реализовано:**

- единый transport-layer для Telegram и VK
- унифицированные модели `Message`, `Event`, `Keyboard`, `Button`, `Attachment`
- wrapper-сообщения: одно бизнес-сообщение может отправляться, редактироваться и удаляться как единая сущность, даже если в разных платформах оно разбивается по-разному
- расширяемая архитектура для добавления новых платформ через адаптеры
- пользовательские сессии и state-machine сценариев
- сценарий записи клиентов на занятия
- административные сессии для управления пользователями, администраторами, филиалами, датами и настройками
- связь клиент ↔ администратор через бота, в том числе между разными платформами
- GPT-ответы клиентам через OpenAI-compatible API
- настройка моделей, контекстов, вопросов и лимитов нейросети прямо через интерфейс бота
- подсчёт токенов и контроль расходования лимитов
- очереди сообщений и фоновых задач
- SQL-хранилище через SQLAlchemy
- миграции схемы через Alembic
- локальный запуск на SQLite
- Docker Compose-окружение с PostgreSQL
- unit-тесты для transport-layer, storage-layer, callback-ов, меню, сессий, GPT и очередей
- CI-проверки: компиляция проекта, import sanity check, миграции и запуск тестов

**Особенность проекта:**  
бот реализован не как набор отдельных handlers под одну платформу, а как расширяемая bot-платформа с общей моделью сообщений, сессий, администрирования, GPT-настроек и SQL-хранилищем.
