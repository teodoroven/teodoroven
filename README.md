## Featured Projects

### Edo — автоматизация ЭДО и OCR документов

Система для автоматизации электронного документооборота. Проект обрабатывает документы из СБИС по API, а также бумажные документы и PDF/сканы через OCR. Результаты синхронизируются в Google Таблицы, чтобы заказчик мог работать с данными вручную и видеть актуальное состояние.

**Стек:**  
Python, СБИС API, PaddleOCR, Tesseract, PyMuPDF, Pillow, SQLite, Google Sheets API, ThreadPoolExecutor, PyInstaller, unittest

**Что реализовано:**

- интеграция со СБИС API
- загрузка входящих и исходящих документов
- OCR-обработка PDF и сканов
- классификация бухгалтерских документов
- извлечение ИНН, дат, номеров и статусов
- проверка дубликатов
- локальное хранение состояния в SQLite
- синхронизация результата в Google Таблицы
- пакетные обновления через Google Sheets API
- обработка ошибок и повторные попытки при сбоях
- упаковка приложения в standalone `.exe`
- тесты для ключевых частей логики и OCR-сценариев

**Особенность проекта:**  
важен не только внутренний результат программы, а актуальная информация в Google Таблице, с которой продолжает работать заказчик.

Desktop-приложение для автоматического мониторинга входящих писем по IMAP, анализа писем по пользовательским правилам и записи результата в Google Таблицы.

**Стек:**  
Python, Eel, Vanilla JavaScript, HTML, LESS, IMAP, Google Sheets API, threading, pystray, tkinter, unittest

**Что реализовано:**

- подключение к почте через IMAP
- анализ тем и текстов писем по пользовательским триггерам
- извлечение дат, контрагентов, периодов и типов отчётности
- запись результата в Google Таблицы
- цветовое кодирование и комментарии в ячейках
- графический интерфейс на Python + Eel
- фоновая работа через системный трей
- хранение и шифрование настроек
- многопоточная синхронизация без блокировки интерфейса
- обработка ошибок сети и внешних API
- тесты для ключевой бизнес-логики

**Особенность проекта:**  
Google Таблица используется как основное рабочее пространство заказчика: пользователь может вносить изменения вручную, а программа учитывает актуальное состояние таблицы.

### Bots — Telegram/VK bot framework для автошколы

Мультиплатформенный бот для автошколы с поддержкой Telegram и ВКонтакте, LLM-ответами, пользовательскими сессиями, административной логикой и связью клиента с администратором через разные платформы.

**Стек:**  
Python, Telegram Bot API, VK API, OpenAI/LLM API, threading, JSON storage, unittest

**Что реализовано:**

- единая архитектура для Telegram- и VK-ботов
- унифицированные модели `Message`, `Event`, `Keyboard`, `Attachment`
- возможность добавления новых платформ через адаптеры
- пользовательские сессии и state machine
- запись клиентов на занятия
- административные сессии для управления данными
- связь клиент ↔ администратор даже через разные платформы
- LLM-ответы клиентам
- настройка контекстов и лимитов нейросети через интерфейс бота
- контроль использования моделей
- обработка голосовых сообщений
- файловое JSON-хранилище

**Особенность проекта:**  
бот реализован не как один сценарий на одной платформе, а как расширяемая система с общей логикой сообщений, состояний и администрирования.
